軟體技術日新月異，如何確定選擇投入的領域是正確有回報的？

工業工程學有幾項重要學科，其中兩項是統計與決策分析。

經驗上，將根據相關調查結果，並依據自身需求決策。

  1. 職缺調查：有職缺表示有需求，有需求表示該領域有人投注資源。
  2. 科技趨勢：大趨勢會導致相關需求誕生，例如 AI。

藉由相關調查結果，加入自身的資料統計分析，以決策出當下對自己有利的選擇。

請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

於矽品工作時，因組織變動而有多頭馬車的情況。一份報告會有多位主管回饋，使底下的工程師混亂。

而我在初次經歷時也感到不知所措，前期的每次討論，負面情緒不減反增。 後來透過剖析問題與經驗解決問題與抒發情緒。

  1. 剖析問題：深入研究問題的核心，是預防、解決還是政治操作，調整報告內容。
  2. 經驗：透過相關信件、需求或之前的報告，找出最高機率是哪位主管會使用，調整成其喜好的模式。

了解這兩項之後，逐漸轉化負面情緒為動力，激勵自己要表現更好給「最高機率是哪位主管」的該主管知道。

讓其了解到我所做的報告是可以符合需求，且優於其他工程師，進而使該主管協助我。

透過類似的處理方式，除了能將負面能量轉化為成就感，更可以展現自我價值。

關於這份申請網頁，分享一個開發時的技術心得。

研究所的必修課碰過 HTML、CSS 與 PHP，並做出過三個專案。

但這次深入學習後發現有很多更專業的方式，像是可以使用 Sass 幫助開發，不用像研究所時一步一步更改 CSS 檔。現在可以更快處理 CSS 程式碼，像是使用變數，在更改或最佳化程式碼時更方便。

如何看待自身工作和整個社會群體的連結關係？

必須提到【野球革命】團隊。

野球革命建構網站，幕後團隊分析各種棒球數據，讓大眾可以免費使用棒球的基本資料，這些資料可以使球迷討論更加熱絡。更不用說這次棒球十二強賽事，野球革命公告限時免費開放多位國外棒球選手的進階資料，對於社會的貢獻不在話下。

有野球革命的示範，更能了解到軟體工程師與社會的連結是互惠互利的。有需求就有供給，進而帶出相關使用。有了更多使用將會創造更多需求，這就是我認為的互惠互利循環。
